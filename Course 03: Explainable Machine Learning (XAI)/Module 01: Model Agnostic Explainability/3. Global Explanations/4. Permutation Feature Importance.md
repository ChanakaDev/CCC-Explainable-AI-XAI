# Permutation Feature Importance

вЂ‹The importance of a feature can be measured by calculating how much model'sВ 
вЂ‹prediction error increases after permuting the feature.В 
вЂ‹If shuffling a feature's value increases the model's error,В 
вЂ‹the feature is important.В 
вЂ‹If the model error doesn't change after shuffling a feature's values,В 
вЂ‹a feature is considered unimportant.В 
вЂ‹This concept was first introduced for random forests in 2001.В 
вЂ‹It was updated to be model agnostic and renamed model reliance in 2018.В 
вЂ‹Let's walk through the model agnostic method developed in 2018 calledВ 
вЂ‹model reliance.В 
вЂ‹Our input is the trained model f, hat, feature matrix x,В 
вЂ‹target vector y, and an error measure.В 
вЂ‹First, we estimate the original model error.В 
вЂ‹The mean squared error is commonly used here.В 
вЂ‹Then, for each feature,В 
вЂ‹we generate feature matrix x perm by permuting feature j in the data x.В 
вЂ‹This breaks the association between feature j and the true outcome y.В 
вЂ‹Next, we estimate the error based on the predictions of the permuted data.В 
вЂ‹We calculate the permutation feature importance either as a quotient orВ 
вЂ‹a difference.В 
вЂ‹Lastly, we sort features by descending feature importance.В

вЂ‹Let's evaluate the pros and cons of permutation feature importance orВ 
вЂ‹model reliance as an explainable AI technique.В 
вЂ‹This method is easy to interpret.В 
вЂ‹It provides global insight into model behavior, andВ 
вЂ‹it does not require retraining the model.В 
вЂ‹However, because the method depends on shuffling the feature,В 
вЂ‹which adds randomness to the measurement when repeated, the results might vary.В 
вЂ‹If features are correlated, it can be biased by unrealistic data instances.В 
вЂ‹There are alternative ways of calculating feature importance.В 
вЂ‹Model specific feature importance examples include the Gini importance in randomВ 
вЂ‹forests and standardized regression coefficients in regression models.В
