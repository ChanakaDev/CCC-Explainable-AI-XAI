# XAI in Knowledge Augmentation (RAG)

## What is knowledge augmentation?

**Retrieval-Augmented Generation (RAG)** is the most common form of LLM knowledge augmentation. Real-world examples:

- **CIONIC chatbot** — customized chatbot.
- **Rent the Runway** — AI search.
- **OpenAI Assistants API** — built on a RAG pattern.

## Embeddings — quick review

We need to convert human concepts (text, images) into numbers a machine can process. We do this by training on lots of data so that **semantically similar concepts end up close together** in the resulting vector space.

Example — context disambiguates meaning:

- "bank" + river context → close to *shore*, *riverside*.
- "bank" + money context → close to *financial institution*.

**Embeddings** = vectors of real numbers that capture semantic and syntactic aspects of data.

Key properties:

- **Dimensionality reduction** — squeeze rich concepts into a fixed-length vector.
- **Semantics** — encode meaning based on usage and context.
- **Distance measures** — similarity in meaning correlates with proximity in vector space.

The space embeddings live in is called the **embedding space** or **latent space**. With multimodal embedding models like **CLIP**, multiple modalities (e.g. images + captions) can share one space — clusters often correspond to human-recognizable concepts (people, clothes, housing, art, animals, etc.).

## The RAG pipeline

### 1. Vector database

Take unstructured data (PDFs, images, videos) and embed it with an **embedding model**. Store the resulting vectors in a vector database.

- The embedding model fixes the **vector length / dimensionality**.
- Example: OpenAI's `ada-002` produces 1536-dim vectors.

### 2. Embed the user query

When the user sends a query, embed it with **the same embedding model** used to build the database.

> Example: query *"black leather boots"* → vector embedding of that phrase.

### 3. Similarity search

Match the query embedding against the vector database using a similarity algorithm.

**Cosine similarity** is the standard choice in RAG, because it is:

- **Scale-invariant** — measures similarity regardless of vector magnitude.
- **High-dim friendly** — focuses on angle rather than absolute distance, which avoids issues from the curse of dimensionality.
- **Interpretable** — values are bounded in [−1, 1].

(Euclidean distance and others can be used, but are less common.)

### 4. Generation

Feed the closest-matching items, along with the user query, into the LLM as context. The LLM produces the final response.

### Recap

> Vector DB ← embed unstructured data → user query (embedded with the same model) → similarity search → top matches injected into the prompt → LLM generates → response to user.

Real-world RAG systems often add complexity:

- **Agentic** structures with an LLM in the loop making decisions.
- **Multiple databases**.
- **Reranking algorithms** that re-order similarity results before passing them to the generator.

## RAG as a source of explainability

RAG is not inherently an XAI method, but the **source grounding** it provides gives users:

- A way to **verify factual accuracy** (you can see what was retrieved).
- Insight into the model's **reasoning process**.
- Confidence that responses are grounded in real data, not hallucinated.

This transparency builds trust and makes LLM capabilities more actionable.

## Making the RAG system itself more explainable

A natural next step: how do we explain the RAG pipeline? One major approach is **visualizing the embedding space**.

### Why visualization is hard

- A 2D plot has dimensions x, y.
- A 3D plot adds z.
- Real embeddings have **1,000+ dimensions**.

We can't directly perceive 1,000-dimensional space, so we need **dimensionality reduction**.

### Three standard techniques

| Method | What it captures | When to use |
|---|---|---|
| **PCA** (Principal Component Analysis) | Global, **linear** relationships | Simplify data, find global linear patterns |
| **t-SNE** (t-distributed Stochastic Neighbor Embedding) | **Local** structure — places similar points close | Visualize local clusters, reveal patterns |
| **UMAP** (Uniform Manifold Approximation and Projection) | **Non-linear** manifold structure | Preserve local structure, handle complex non-linear relationships |

These are commonly used in tools like the **TensorFlow Embedding Projector** to interactively explore embedding spaces (covered in the next reading).
