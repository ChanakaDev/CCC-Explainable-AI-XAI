# XAI in LLM Fine-tuning

## Quick review: what is fine-tuning?

A **base model** (typically >1B parameters) is pre-trained on a large unlabeled text corpus via unsupervised learning. Base models are usually not good at producing useful responses on their own.

To produce a useful **assistant model**, the base model is adapted on a smaller dataset using:

- **Supervised fine-tuning (SFT)** with labeled data, and/or
- **Reinforcement Learning from Human Feedback (RLHF)**.

Example pipeline: large text corpus → unsupervised pre-training → LLAMA-2 base → SFT/RLHF → LLAMA-2 chat.

## Two families of explanations

| Type | What it explains |
|---|---|
| **Local** | A specific prediction or set of predictions |
| **Global** | The model's behavior as a whole |

---

## Local explanations

Four sub-categories: feature attribution, attention-based, example-based, and natural language.

### 1. Feature attribution

Four primary approaches:

- **Perturbation-based.** Remove, mask, or alter input features (embedding vectors, hidden units, words, tokens) and observe how the output changes.
- **Gradient-based.** Compute partial derivatives of the output with respect to each input dimension. Magnitude indicates sensitivity. **Integrated Gradients** is the dominant method for LLMs.
- **SHAP.** Game-theoretic feature attribution. **TransSHAP** (Chen et al., 2023) adapts SHAP to transformer-based language models, focusing on sub-word text input and sequential visualizations.
- **Decomposition-based.** Break the relevance score into linear contributions from each input. Example: **Layer-wise Relevance Propagation (LRP)**.

### 2. Example-based explanations

- **Counterfactual explanations** — what would have happened if the input changed in a specific way.
- **Influential instances** — measure how much individual training samples affect the loss on a given test point.
- **Adversarial examples** — see below.
- **Natural language explanations** — see below.

### 3. Adversarial examples

Small, often imperceptible input modifications that drastically alter model predictions. They:

- Expose model failure modes.
- Can be used during training to improve robustness.

Evolution of methods:

| Generation | Approach |
|---|---|
| Early | Word-level: typos, removals, insertions |
| **TextFooler** (Jin et al., 2020) | Strategically perturb important words using word embeddings + semantic similarity |
| State-of-the-art | Use masked language models with **mask-then-infill** for token-level replacement, insertion, merging |
| **SemAttack** (Wong et al., 2022) | Operates directly on embedding spaces (knowledge, contextualized semantics); iteratively optimizes perturbed embeddings to meet attack goals |

A typical attack swaps a word for an adversarial substitute that flips the model's prediction without fooling a human reader.

### 4. Natural language explanations

Train a language model on both the original textual data **and** human-annotated explanations, so it can generate text justifying its decisions.

- Low et al. (2019) introduced the **Common Sense Explanations (CoS)** dataset and used it to train models to generate explanations at training and inference time.
- A 2022 study explored **few-shot, out-of-domain transfer learning** of natural language explanations from a parent task (rich in NL explanations) to a child task.

## Global explanations

Three sub-categories: concept-based, probing-based, and neuron activation.

### 1. Concept-based explanations

Map inputs to a set of predefined concepts and measure how important each concept is to the model's prediction. Example: **Testing with Concept Activation Vectors (TCAV)**.

### 2. Probing-based explanations

Two flavors:

**Classifier-based probing**

1. Freeze the LLM's parameters.
2. Generate representations from the LLM.
3. Train a **shallow classifier** on those representations to predict linguistic properties.

What you probe for:

- **Syntax** — parts of speech, morphology, dependencies.
- **Semantics** — coreferences, entities, relations.

**Parameter-free probing.** Evaluates an LLM directly on tailored datasets, with no auxiliary classifier.

> **Caveat:** Probing validity is debated. High classifier performance does not necessarily mean the LLM truly "understands" the linguistic property — it may just mean the representation is rich enough for a shallow classifier to extract the signal.

### 3. Neuron activation explanations

Analyzes **individual neurons / dimensions** rather than the whole vector space. Two-step structure:

1. **Identify important neurons.**
2. **Learn the relationship** between linguistic properties and those neurons.

Example: visualizing the activation of the 131st neuron in the 5th layer of GPT-2.

Full pipeline:

1. **Identify important neurons** (unsupervised) — rank them via correlation, learned weights, or supervised classification.
2. **Learn relations** between linguistic properties and ranked neurons in supervised tasks.
3. **Verify importance** quantitatively via ablation experiments (masking, erasure).
4. **Generate natural language explanations** that summarize the text patterns that trigger high activation for a given neuron (often with another LLM).
5. **Evaluate** the NL explanation by testing how well it lets the model **simulate** the real neuron's activation behavior on new examples.

> Evaluation is hard because there is no ground-truth annotation for what each neuron should represent.
